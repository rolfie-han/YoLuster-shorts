# YoLuster Shorts 釉光影 Docker 本地部署包 v2.0.0

YoLuster Shorts v2.0 是一次面向短剧 / 漫剧创作流程的重大升级。

本版本重点强化了「导入剧本 -> 自动解析剧情 -> 生成人物 / 场景 / 物品提示词 -> 分镜生成 -> 图片生成 -> 视频输出」的完整创作链路。目标是让不会写提示词的用户，也能从一份剧本文本开始，逐步生成可用于短剧或漫剧制作的高质量素材。

## v2.0 主要更新

### 1. 剧本自动解析能力大幅增强

- 支持导入剧本后自动解析多集剧情内容。
- 自动提取人物、场景、物品、关系和关键剧情信息。
- 自动生成适合后续生图、生视频使用的提示词。
- 降低提示词门槛，小白用户无需手写复杂 prompt，也能进入完整创作流程。

### 2. 多模型与多中转路线适配

v2.0 接入并适配了更多文本、语音、生图、识图和视频模型路线，方便用户根据自己的 API Key 和服务商能力进行选择。

代表性适配方向包括：

- 文本模型：用于剧本解析、剧情扩写、分镜规划、提示词生成。
- 识图模型：用于素材理解、参考图分析、资产辅助描述。
- 生图模型：用于角色、场景、道具、分镜画面生成。
- 语音模型：用于角色配音、旁白和音频生成。
- 视频模型：用于图生视频、参考图生视频、多图参考视频生成。

代表性视频模型 / 路线适配包括：

- Veo 3.1
- Sora 2
- Seedance 2.0
- Kling 3.0
- Grok 3 Video
- Vidu / Vidu Q3
- MiniMax Hailuo
- WAN2.6
- HappyHorse 1.0
- OpenAI-compatible / 自定义中转路线

实际可用模型取决于用户自行配置的 API Key、服务商权限、额度和接口状态。

### 3. 全新视频输出工作流

- 新增独立的视频输出页面。
- 分镜视频生成、等待输出、结果预览更加集中。
- 视频生成链路更清晰，便于管理多镜头任务。
- 优化分镜到视频输出之间的衔接体验。

### 4. 创作工作台体验升级

- 优化专业创作工作台结构。
- 强化剧本解析、素材库、分镜脚本、视频生成、结果预览之间的流程边界。
- 单图创作、多图叙事、视频生成等高级工具区更加独立清晰。
- 优化素材选择、上传、删除和页面状态同步体验。

### 5. 登录页与品牌视觉升级

- 重新优化登录页视觉表现。
- 加入更强的品牌 Logo 展示。
- 优化右侧视频 / 图片展示区，让首页观感更接近正式产品。
- 背景视频增加视觉遮罩，减少干扰，提升登录页整体质感。

### 6. 并发与任务体验优化

- 优化多个功能区的并发处理能力。
- 改进长任务执行中的反馈体验。
- 减少生成流程中的卡顿感和状态不明确问题。
- 强化素材、分镜、视频任务之间的状态同步。

### 7. 本地 Docker 部署包优化

- 提供更适合公开分发的 Docker 本地部署薄包。
- 本地包仅包含运行所需的前端静态资源、Docker 启动文件和代理配置。
- 不包含 YoLuster Shorts 云端后端核心代码。
- 适合 GitHub 用户下载后在本地 Docker 环境中运行。

### 8. 账号与基础合规流程完善

- 注册流程增加用户协议与隐私政策确认。
- 完善邀请注册码、密码找回等账号基础能力。
- 强化本地部署场景下的使用边界说明。
- API Key 由用户自行配置和管理，请遵守对应模型服务商的使用规则。

## 使用方式

1. 下载本版本中的 `yoluster-shorts-v2.0.0-yoluster-brand-sync-docker-local-package-thin-bundle.zip`。
2. 解压到本地英文路径，例如：`D:\yoluster-shorts`。
3. 安装并启动 Docker Desktop。
4. 按压缩包内 README 或启动脚本运行。
5. 在浏览器打开本地地址使用。

如果你已经部署过旧版本，建议先备份旧目录和个人配置，再解压新版重新启动。

## 注意事项

- 本地 Docker 包不包含云端后端核心代码。
- 用户需自行准备第三方模型 API Key。
- 模型调用费用、额度、封禁、价格变动由对应服务商负责。
- 首次部署建议让 AI 助手根据 README 协助执行。
- 如遇端口占用、Docker 未启动、WSL2 未开启等问题，请参考包内说明排查。
