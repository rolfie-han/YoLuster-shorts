# Releases

这里会放 `YoLuster Shorts 釉光影` 的公开体验包和演示材料。

如果你想先在自己的电脑上看一看产品形态、界面节奏和创作流程，可以从 Release 里下载本地软件包。它的重点不是让你折腾部署，而是尽快跑起来、点进去、看清楚这套 AI 短剧工作台到底在解决什么问题。

## v2.0.0

这是一次面向短剧 / 漫剧完整创作流程的重大升级。

v2.0 强化了「导入剧本 -> 自动解析剧情 -> 生成人物 / 场景 / 物品提示词 -> 分镜生成 -> 图片生成 -> 视频输出」的完整链路。它更适合希望从剧本文本出发，而不是从复杂提示词出发的用户。

核心变化：

- 剧本自动解析能力增强，支持多集剧情、人物、场景、物品和关键剧情信息提取。
- 自动生成后续生图、生视频所需的结构化提示词，降低小白用户上手门槛。
- 接入并适配更多文本、语音、生图、识图和视频模型路线。
- 新增独立视频输出工作流，分镜视频生成、等待输出和结果预览更加集中。
- 优化专业创作工作台、素材状态同步、上传删除体验和多个功能区的并发能力。
- 优化登录页与品牌视觉，增强首页展示质感。
- 完善注册协议确认、邀请注册码、密码找回等基础账号流程。
- 提供更适合公开分发的 Docker 本地部署薄包。

代表性视频模型 / 路线适配包括 Veo 3.1、Sora 2、Seedance 2.0、Kling 3.0、Grok 3 Video、Vidu / Vidu Q3、MiniMax Hailuo、WAN2.6、HappyHorse 1.0 以及 OpenAI-compatible / 自定义中转路线。

实际可用模型取决于用户自行配置的 API Key、服务商权限、额度和接口状态。

## v0.1.1-docker-thin-public-clean

这是第一版 Docker 本地体验包。

适合你在这些场景下使用：

- 想快速看看 YoLuster Shorts 的整体界面和创作入口。
- 想体验从创作工作台进入脚本、图片、视频相关流程的产品结构。
- 不想一上来配置复杂模型服务和一堆 API。
- 想给朋友、用户或团队成员一个更直观的试用入口。

### 你可以体验到什么

- 一个更接近真实产品的本地运行入口。
- 釉光影的创作工作台界面和主要功能区组织方式。
- 从想法到脚本、图片、视频相关流程的产品连接感。
- 面向普通创作者的使用路径，而不是面向工程师的接口拼装页。

### 小白怎么开始

1. 安装并启动 Docker Desktop，或确认本机已有可用的 Docker 环境。
2. 下载 `docker-local-package-thin-bundle.zip` 并解压。
3. 把 `.env.example` 复制一份，改名为 `.env`。
4. 在 `.env` 里填写官方提供的 `CLOUD_BACKEND_ORIGIN`，如果 `8080` 端口被占用，可以修改 `LOCAL_WEB_PORT`。
5. 在解压目录运行：

```powershell
docker compose up -d --build
```

启动后打开：

```text
http://127.0.0.1:<LOCAL_WEB_PORT>
```

停止时运行：

```powershell
docker compose down
```

### 小提示

这个包是面向公开体验的本地软件包，不是完整源码自部署版。核心生成能力和服务调度由官方服务提供，所以你不需要在本地维护复杂模型、密钥和后端环境。

如果你想先快速了解产品能力，可以先看仓库首页的截图、案例和演示视频；如果你想实际跑起来，请下载本页的官方 zip 包。

### 包内容说明

这个体验包只包含本地启动和体验所需文件，不包含你的个人配置。实际使用前，请按说明复制 `.env.example` 为 `.env`，并填写官方提供的测试配置。
